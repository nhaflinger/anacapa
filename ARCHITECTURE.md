# Anacapa Renderer — Architecture & Design

## Overview

Anacapa is a physically-based offline renderer built in C++. It provides two co-equal CPU integrators: a **unidirectional path tracer** (PathIntegrator) and a **bidirectional path tracer** (BDPTIntegrator) with Veach MIS. Unidirectional path tracing is the primary focus for production work; BDPT serves as a complement for scenes where it offers variance advantages. Both are backed by a **SAH BVH** for ray traversal and **Halton low-discrepancy sampling**. Optional **Metal / CUDA GPU backends** implement a unidirectional path tracer for interactive preview. Scenes are loaded from **USD/USDA/USDC** files. Materials are **MaterialX / OpenPBR** networks, evaluated via **OSL shaders** generated at export time. Output is HDR EXR with optional ACES-tonemapped PNG and optional **Intel OIDN denoising**.

## Scene Loading

`USDLoader` (src/scene/usd/USDLoader.cpp) traverses the USD prim hierarchy and constructs all scene data.

### Geometry → `GeometryPool`

Each `UsdGeomMesh` becomes a `MeshDesc` with parallel SoA arrays: positions, normals, UVs, indices. **SoA (Structure of Arrays) layout is intentional** — GPU porting is trivial (each array → one buffer) and linear traversal is cache-friendly.

**Motion blur:** Multi-key transforms are stored in `MeshDesc::motionKeys`. At intersection time, the world transform is linearly interpolated at `ray.time`. Vertices for animated meshes are kept in object space (`isObjectSpace=true`) and transformed per-hit. BVH bounds at build time are the conservative union of all key-frame bounds.

Z-up to Y-up correction is applied via `applyUpCorrection()` for Z-up USD scenes.

### Lights → `ILight` subtypes

| USD prim | Anacapa type |
|---|---|
| `UsdLuxRectLight` | `AreaLight` (quad with half-extents u/v) |
| `UsdLuxSphereLight` | `AreaLight` (approximated as quad) |
| `UsdLuxDistantLight` | `DirectionalLight` |
| `UsdLuxDomeLight` | `DomeLight` (equirectangular HDRI) |

Intensity is resolved as `intensity × 2^exposure` (USD standard).

### Camera → `Camera` struct

Pinhole or thin-lens (if fstop specified). Selection priority: explicit `--camera` prim path > UsdRenderSettings.camera > first camera found > auto-framed bounds camera.

### Materials → `IMaterial` subtypes

USD materials are walked from `UsdPreviewSurface` / `ND_open_pbr_surface_surfaceshader`. Textures are loaded via OpenImageIO and stored as TOV (Texture-Or-Value) fields allowing per-pixel evaluation. OSL materials are compiled at runtime from `.osl` files generated by the Blender export script.

### Fallback Scene

If no USD file is provided, a hardcoded Cornell box is built: 6 quad walls, 2 rotated interior boxes (classic angles −15° and +18°), and a ceiling area light with emissive radiance 15 W/sr.

### Post-load Overrides

`--override-lights`, `--override-materials`, `--light-angle`, `--fstop`, auto-sun injection if no lights found.

## Rendering Pipeline — `RenderSession::render()`

### Build Phase

1. `buildAccelStructure()` — commits the SAH BVH over the full `GeometryPool`
2. `DomeLight::loadImage()` — loads HDRI; builds 2D importance sampling distribution
3. Creates `Film`, `IIntegrator`, `HaltonSampler`, `ThreadPool`
4. `integrator->prepare(scene)` — builds `LightSampler` CDF, GPU buffer uploads

### Tile Scheduling

The film is partitioned into `tileSize × tileSize` tiles (default 64×64). Each `TileRequest` carries `(x0, y0, width, height, sampleStart, sampleCount)`. Tile workers are completely independent — no shared mutable state during rendering.

### Render Dispatch

**Non-adaptive:** Attempts `renderFrame()` (GPU whole-image dispatch first). Falls back to per-tile `parallelFor()` if GPU unavailable.

**Adaptive (two-pass):**
- *Base pass:* `baseSpp = min(32, max(16, spp/16))` samples over the full image
- *Variance computation:* Per-tile variance from `Film::varianceAt()` → `E[L²] − E[L]²`
- *Refinement pass:* Remaining `spp − baseSpp` samples allocated as:
  - **50% uniform share** — every tile receives an equal floor
  - **50% proportional** — weighted by `log(1 + tileVariance)`

  The blend ensures every tile gets minimum refinement while concentrating extra samples on noisy regions.

**GPU path:** `renderFrame()` dispatches the entire image as a single compute kernel. A persistent accumulation buffer avoids re-allocation. `clearAccum()` is called before each fresh render (scene or camera change).

### Progressive Preview

A dedicated thread wakes every 500 ms. If `Film::m_dirty` is set (by `mergeTile()`), it writes a tonemapped PNG. Uses a condition variable for clean shutdown. Render workers are never stalled by disk I/O.

## CPU Integrators

### `PathIntegrator` — Unidirectional Path Tracing

The primary integrator for production work. Unidirectional path tracing is a better fit for production pipelines: predictable performance, straightforward debugging, and compatibility with standard rendering techniques (light path expressions, AOVs, denoising). BDPT is a complement, not a replacement.

**Per-path loop:**
1. Intersect ray → `SurfaceInteraction` via BVH
2. Stochastic alpha test (`rand < opacity` → continue ray through surface)
3. Accumulate emission `Le` (first bounce only, or after specular bounce, to avoid double-counting with NEE)
4. **Direct lighting (NEE):** Sample one light uniformly; MIS-weight against BSDF
   - Light sampling branch: sample direction from light, check occlusion
   - BSDF sampling branch: sample BSDF, check light PDF
   - `powerHeuristic(pdfA, pdfB) = pdfA² / (pdfA² + pdfB²)`
5. Sample BSDF for next bounce direction; multiply throughput β
6. **Russian roulette** after depth 2: terminate with probability `1 − clamp(max(β), 0, 0.95)`

### `BDPTIntegrator` — Bidirectional Path Tracing

Complementary integrator. Gives better variance for indirect lighting, caustics, and glossy inter-reflections where unidirectional path tracing converges slowly. The direction of travel is toward using BDPT selectively to augment the unidirectional approach rather than as a wholesale replacement.

**Algorithm:** Veach 1997, §10.3.

**Per-pixel sample:**
1. Sample frozen scene time `t ∈ [shutterOpen, shutterClose]` — used for all bounces
2. Trace **camera subpath** (up to `maxDepth+2` vertices) forward from camera
3. Trace **light subpath** (up to `maxDepth+2` vertices) forward from a sampled light
4. For each `(s, t)` strategy (s light vertices, t camera vertices):
   - Validate: not both delta surfaces, not mutually occluded, roughness threshold check
   - Compute unweighted contribution `C(s,t)`
   - Clamp to `fireflyClamp` luminance before MIS weighting (weight ∈ [0,1] so pre-clamp is correct)
   - Compute MIS weight `w(s,t)`
   - Accumulate `w(s,t) × C(s,t)` into film (or splat light-endpoint strategies directly to their projected pixel)

**`PathVertexBuffer` — SoA layout:**

```
position[]          — world-space vertex position
normal[]            — shading normal
wo[]                — outgoing direction (toward previous vertex)
beta[]              — path throughput to this vertex
Le[]                — emitted radiance (non-zero at light endpoints)
pdfFwd[]            — area-measure PDF of sampling from previous vertex
pdfRev[]            — area-measure PDF of sampling previous from this vertex
flags[]             — Camera | Surface | Light | Delta | Infinite bits
meshID[], material[], light[]
pathMinRoughness[]  — minimum roughness seen so far along the path
sceneTime           — frozen scene time for all bounces
lightPdf            — probability of selecting this light
```

All PDFs are stored in **area measure** throughout — no per-step unit conversion is needed in the MIS computation.

**MIS weight** uses the recursive ratio method:

```
r_fwd[i] = p(s−i, t+i) / p(s,t)   — shift connection one step toward camera
r_rev[i] = p(s+i, t−i) / p(s,t)   — shift connection one step toward light
w(s,t)   = 1 / (1 + Σ r_fwd[i]² + Σ r_rev[i]²)
```

Each ratio is computed by multiplying/dividing consecutive `pdfFwd`/`pdfRev` values — O(depth) per strategy.

**Frozen scene time:** `sceneTime` is sampled once per pixel and reused for every intersection in both subpaths. Ensures a moving object occupies one consistent position across the full path, preventing spatial artifacts from per-bounce time variation.

**`pathMinRoughness`:** Tracks the minimum material roughness seen along the path so far. Connections involving a near-specular surface at both ends are skipped (geometry PDF ≈ 0, contribution negligible, numerical instability).

## Shading & Materials

### `IMaterial` Interface

```cpp
BSDFSample  sample(ctx, wo, u2, uComponent);  // Sample wi; return f, pdf, pdfRev, flags
BSDFEval    evaluate(ctx, wo, wi);            // f(wo,wi), pdf, pdfRev
float       pdf(ctx, wo, wi);
Spectrum    Le(ctx, wo);                      // Emission
bool        isDelta();                        // Specular/delta vs diffuse
float       roughness(), metalness();         // Metadata for GPU extraction
```

All methods are const and stateless. Per-evaluation context lives in `ShadingContext`. `pdfRev` (reverse PDF) is returned by `sample()` and `evaluate()` for BDPT MIS — avoids a redundant separate evaluation.

### Material Types

**`LambertianMaterial`**
- f = ρ/π, cosine-weighted hemisphere sampling, pdf = cosθ/π
- Used as the `--override-materials` fallback and Cornell box walls

**`StandardSurfaceMaterial`**
- Multi-layer BSDF approximating MaterialX OpenPBR
- Layers evaluated in order:
  1. **Glass/transmission** (if transmission > 0): exact Fresnel + Snell for smooth glass (roughness < 0.001); GGX microfacet transmission (Walter 2007) for rough glass
  2. **Coat** (if coat > 0): thin dielectric GGX on top, IOR=1.5
  3. **Specular**: GGX reflection; Schlick Fresnel for dielectric; colored F0 for metallic
  4. **Diffuse base**: Lambertian with base_color
- All parameters support **TOV (Texture-Or-Value)** — constant or per-pixel texture lookup
- Normal mapping: perturbs shading normal in tangent space

**`OslMaterial`**
- Executes an arbitrary OSL shader network (`.osl` → compile → `.oso` → `oslexec`)
- Used for the full procedural Blender material graphs (noise, color ramps, math chains, etc.)
- Compilation happens at load time via `oslcomp`; execution per shading point via `oslexec`

**`EmissiveMaterial`**
- Returns constant `Le`; not sampled directly (only via NEE / BDPT light subpath)

### Light Types

**`AreaLight`**
- Planar quad defined by center and two half-extent vectors (u/v axes)
- Uniform area sampling; solid-angle PDF = `dist² / (area × cosθ)`
- For BDPT `sampleLe()`: uniform position on quad, cosine-weighted emission direction

**`DirectionalLight`**
- Infinite distance, parallel rays (sun-like); `isDelta=true`, `isInfinite=true`
- **Soft shadows** via cone jitter: `--light-angle` sets a half-angle; shadow rays are jittered within the cone. `isDelta=true` is preserved so MIS weight stays 1.0 — no MIS penalty, correct energy
- For BDPT `sampleLe()`: imaginary disk at `100 × sceneRadius` from scene center in light direction, giving a finite ray-spawn point with negligible angular error

**`DomeLight`**
- Equirectangular HDRI loaded via OpenImageIO
- **sRGB→linear decode** applied at load time for LDR formats (JPEG, PNG, TIFF); EXR/HDR are kept linear
- Builds a 2D importance sampling distribution: pixels weighted by `luminance × sin(θ)` (solid-angle equalization), with per-row CDFs for fast 2D inversion sampling
- Supports world-space rotation via `setRotation()` (3×3 world-to-envmap matrix) for USD DomeLight xform
- For BDPT `sampleLe()`: spawn point on a disk perpendicular to sampled direction at `sceneCenter + direction × sceneRadius`

## Acceleration Structure — SAH BVH

### Node Layout (32 bytes, depth-first)

```cpp
struct BVHNode {
    float    boundsMin[3];
    uint32_t dataA;   // interior: rightChildIndex;  leaf: primOffset
    float    boundsMax[3];
    uint32_t dataB;   // interior: splitAxis;         leaf: primCount | kLeafFlag
};
// Left child is always at nodeIndex+1 — no explicit left-child pointer
```

Depth-first node ordering gives cache-friendly traversal: the left subtree is always contiguous in memory.

### Triangle Storage

```cpp
struct BVHTriangle {
    Vec3f v0, e1, e2;              // Vertex + precomputed edge vectors
    Vec3f n;                       // Geometric normal (normalized)
    Vec2f uv0, uv1, uv2;           // Per-vertex texture coordinates
    Vec3f sn0, sn1, sn2;           // Per-vertex shading normals
    uint32_t meshID, primID;
    bool isObjectSpace;            // True for animated meshes (motion blur)
};
```

### Build Algorithm

SAH with 12 buckets, max 4 primitives per leaf. For each candidate split:

```
cost = kTraversalCost + kIntersectCost × (leftArea × leftCount + rightArea × rightCount) / parentArea
```

All three axes are tested; the minimum-cost split is chosen. A leaf is created if no split beats the leaf cost. After choosing axis and split position, primitives are partitioned in-place.

### Traversal

**AABB intersection:** Slab method with precomputed `invDir` and `dirNeg` arrays for branch-free min/max selection.

**Triangle intersection:** Möller-Trumbore — computes barycentric coordinates (u, v), checks `u ≥ 0, v ≥ 0, u+v ≤ 1`, returns t and surface interaction.

**Traversal loop:** Stack-based DFS. AABB hit → push children. Leaf → intersect all primitives, update closest hit.

**Motion blur at traversal:** When a leaf triangle has `isObjectSpace=true`, the local-to-world transform is linearly interpolated between motion keys at `ray.time` and applied to transform the intersection result.

## Sampling

### `HaltonSampler`

Low-discrepancy sampling via the Halton sequence with per-pixel Owen scrambling.

- Dimension `d` uses prime base `primes[d]` (first 128 primes)
- Per-pixel scramble seed derived from `(pixelX, pixelY)` via PCG-32
- `scrambledRadicalInverse(dim, sampleIndex)`: XOR-scramble the sample index in base `primes[dim]`, then compute radical inverse

```
Radical inverse in base b:
  Reverse the digits of n in base b → 0.d_k d_{k-1} ... d_0  (fractional)
```

Per-pixel scrambling breaks inter-pixel correlation (neighboring pixels don't generate the same low-discrepancy sequence) while preserving the within-pixel stratification that reduces noise. Each tile worker receives an independent `clone()` of the base sampler — no inter-thread sharing.

### `PCGRng`

PCG-32 pseudo-random generator used for scramble seeds, Russian roulette decisions, and stochastic opacity tests. `nextFloat()` ∈ [0, 1), `nextUint32()`.

## Film & Output

### `TileBuffer`

Non-atomic per-tile accumulation during rendering. Stores `(r, g, b, weight)` per pixel plus `sumLumSq` for variance. No contention within a tile. Merged into `Film` atomically at tile completion.

### `Film::PixelAccumulator`

Per-pixel accumulation with **relaxed atomics** on `r, g, b, weight, sumLumSq`. Relaxed ordering is correct because render order is irrelevant — only the final sum matters. This is orders of magnitude faster than acquire/release semantics for high-SPP renders.

### Variance Tracking

`Film::varianceAt(x, y)` returns `E[L²] − E[L]²` computed from `sumLumSq` and accumulated weight. Used by the adaptive sampling pass to allocate refinement samples.

### `Film` AOVs

Two additional `PixelAccumulator` arrays hold first-hit **albedo** and **world-space normals**. These are captured in the integrators on the first camera ray bounce and used as auxiliary inputs to Intel OIDN denoising.

### EXR Output (OpenImageIO)

Multi-layer EXR always includes the beauty layer. Optional: post-OIDN denoised layer, albedo AOV, normals AOV (if `--write-aovs`). Linear HDR float32 — no tone mapping.

### PNG Output

1. Apply exposure adjustment (EV stops)
2. **ACES reference tone mapping:** linear RGB → ACEScc → RRT (reference rendering transform) → Rec.709 ODT (output device transform)
3. sRGB encode → 8-bit PNG

### Intel OIDN Denoising

Uses the `"RT"` (ray-traced) filter. Inputs: beauty + albedo + normals (all float32). The denoised result is stored as a separate layer in the output EXR. Requires `ANACAPA_ENABLE_OIDN`.

## GPU Metal Backend

### Architecture

`MetalPathIntegrator` implements a **megakernel** unidirectional path tracer — the entire bounce loop runs inside a single compute shader thread, keeping all path state in registers. There is no wavefront architecture.

**Why megakernel over wavefront on Apple Silicon:** The unified memory architecture and large register file make in-register bounce loops cheaper than the extra memory bandwidth needed to write and re-read per-ray wavefront state between kernel dispatches. A wavefront implementation was prototyped and measured slower; it was removed.

### `prepare(scene)`

- Extracts all materials into flat `GpuMaterial` structs — no virtual dispatch on GPU:
  ```c
  struct GpuMaterial {
      GpuFloat3  baseColor;
      float      roughness;
      GpuFloat3  emissive;
      float      metalness;
      uint32_t   type;         // kMatLambertian | kMatGGX | kMatEmissive | kMatGlass
      float      specularIOR;
      float      transmission;
  };
  ```
  Material type is detected at prepare time by querying the CPU material's flags and type — `kMatGlass` if transmission > 0, `kMatGGX` if glossy flag set, etc.

- Uploads lights as `GpuLight` tagged unions (`kLightRect`, `kLightSphere`, `kLightDirectional`, `kLightDome`)
- Uploads HDRI as RGBA32Float `MTLTexture` (RGB padded to RGBA row-by-row)
- Creates 1×1 white fallback texture when no dome light is present
- Builds `MetalAccelStructure` (Metal hardware BVH via Metal Raytracing API)

### `renderFrame()` — Batch Sample Packing

Rather than one kernel dispatch per sample, each thread traces `kBatchSize=4` full paths internally and accumulates them in registers before writing once to the accumulation buffer:

```cpp
constexpr uint32_t kBatchSize    = 4;  // paths per thread per dispatch
constexpr uint32_t kMergeInterval = 4;  // dispatches before flushing to Film

for (uint32_t s = 0; s < sampleCount; s += kBatchSize) {
    GpuSampleBatch sb = { sampleStart + s, min(kBatchSize, sampleCount - s) };
    // encode → commit → waitUntilCompleted
    if (++dispatches % kMergeInterval == 0) flushToFilm();
}
```

Batch size 4 is optimal on Apple Silicon: meaningfully reduces command-buffer round-trips without exhausting the register file. Larger batch sizes increase register pressure faster than they reduce CPU/GPU sync overhead.

### `GpuCameraParams` / `SharedTypes.h`

All host↔GPU shared data lives in `SharedTypes.h` (included by both `.mm` and `.metal` files). Key structs: `GpuRay`, `GpuHit`, `GpuMaterial`, `GpuLight`, `GpuCameraParams`, `GpuAccumPixel`, `GpuSampleBatch`.

`GpuFloat3` is `packed_float3` on the Metal side (12 bytes, 4-byte aligned) and `struct { float x, y, z; }` on the C++ side — identical ABI.

### Persistent Accumulation Buffer

`GpuAccumPixel[]` (one per pixel) persists across `renderFrame()` calls. `ensureAccum()` allocates on first call or on size change. `clearAccum()` zeros it before a fresh render. `flushToFilm()` reads back to CPU and merges into `Film`. This design allows progressive refinement: multiple `renderFrame()` calls accumulate additively.

### `Shade.metal` — Shader Kernel

- Per-thread bounce loop (all state in registers, no intermediate buffers)
- Inline GGX, Lambertian, glass BSDFs — no virtual dispatch
- Direct lighting via shadow ray + Metal `intersect()` with `MTLAccelerationStructure`
- HDRI sampled via `texture2d.sample()` with bilinear filtering
- Soft shadows: cone jitter for directional lights, gated by `cosCone` parameter
- PCG-32 random numbers seeded from `pixelIndex ^ (sampleIndex * kGoldenRatio)`
- Threadgroup size: **8×8** = 64 threads = 2 SIMD groups of 32 — optimal for Apple Silicon

## GPU CUDA Backend

Mirrors the Metal backend in architecture: `CudaPathIntegrator`, `CudaAccelStructure`, persistent device accumulation buffer, `kBatchSize=4` batch sample packing, and equivalent quality fixes (soft shadows, glass TIR, mixed diffuse/specular bounce sampler, NEE roughness floor). Developed on a Linux machine; all improvements are kept in sync with the Metal backend.

## OSL / MaterialX Pipeline

### Export Script (`tools/blender_prep_for_usd_export.py`)

Translates the Blender shader node tree for each material into a MaterialX document (`.mtlx`), then uses the MaterialX genosl code generator to produce an `.osl` source file. Each `.osl` file is self-contained:

```
Blender node tree
  → _MtlxBuilder (Python)              — translate nodes to MaterialX graph
  → MaterialX genosl code generator    — generate .osl source
  → oslcomp (at render load time)      — compile .osl → .oso bytecode
  → oslexec (per shading point)        — execute surface closure
```

### `_mx_stdlib.h`

A hand-maintained OSL header that provides implementations of all MaterialX stdlib functions that have no native genosl counterpart. Bundled with every generated shader via `#include "_mx_stdlib.h"`. Includes: `NG_checkerboard_color3`, `NG_place2d_vector2`, `mx_rotate2d`, `mx_normalmap_float`, `mx_image_color3`, `NG_open_pbr_surface_surfaceshader`, and all blend/math/HSV functions.

### Node Translation (`_MtlxBuilder` class)

`_MtlxBuilder` walks the Blender node graph and emits MaterialX nodes:

| Blender node | MaterialX node |
|---|---|
| `BSDF_PRINCIPLED` | `ND_open_pbr_surface_surfaceshader` |
| `TEX_IMAGE` | `ND_image_color3` / `ND_image_float` |
| `TEX_NOISE` | `ND_fractal3d_float` |
| `TEX_CHECKER` | `ND_checkerboard_color3` |
| `VALTORGB` (Color Ramp) | `ND_mix` chain (8-sample baked) |
| `NORMAL_MAP` | `ND_normalmap_float` (via extract+combine3) |
| `MATH` | `ND_multiply_float`, `ND_add_float`, etc. |
| `RGBTOBW` | `ND_luminance` + `ND_extract` |
| `MAPPING` | `ND_place2d_vector2` |
| `SEPARATE_COLOR` | `ND_extract` per channel |

Type coercion: `_socket_float()` calls `_to_float()` on any upstream `color3`/`vector3` output (luminance+extract chain), ensuring arithmetic nodes always receive the correct type.

## Key Architectural Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Primary integrator | PathIntegrator (unidirectional PT) | Better production fit: predictable, debuggable, compatible with standard AOV/denoising pipelines |
| Complementary integrator | BDPTIntegrator | Better variance for caustics and indirect lighting; intended to augment the unidirectional approach selectively |
| MIS strategy | Power heuristic | Robust across diverse lighting; Veach proven optimal for 2 strategies |
| BVH type | SAH with 12 buckets | Standard quality/build-time tradeoff for offline renderers |
| Memory layout | SoA (Structure of Arrays) | Cache-friendly; trivial GPU port (each array → one MTLBuffer / device pointer) |
| Film atomics | Relaxed (`memory_order_relaxed`) | No ordering needed — only final sum matters; far faster than acquire/release |
| Adaptive sampling | 50% uniform + 50% log-variance weighted | Uniform floor guarantees minimum quality; variance weighting concentrates effort |
| Scene time | Sampled once per pixel, frozen for all bounces | Temporal consistency; moving objects at one position for the full path |
| GPU architecture | Megakernel (in-register bounce loop) | Cheaper than wavefront on Apple Silicon (register file > bandwidth) |
| GPU accumulation | Persistent buffer + `clearAccum()` | Avoids reallocation; enables multi-call progressive accumulation |
| GPU batch size | 4 paths per thread per dispatch | Reduces command-buffer round-trips without exhausting register file |
| Soft shadows | Directional cone jitter, `isDelta=true` kept | Free soft shadows — `isDelta=true` preserves MIS correctness (weight stays 1.0) |
| Sampler | Halton + per-pixel Owen scrambling | Low-discrepancy within pixel; independent per-pixel (no inter-pixel correlation) |
| LDR dome lights | sRGB→linear decode at load | OIIO reads float in [0,1] without applying gamma; must decode for correct radiance |
| OSL materials | Compile `.osl` at load time | Full procedural fidelity without baking — Noise, ColorRamp, Math all evaluate analytically |

## End-to-End Data Flow

```
USD file
  └─ USDLoader
       ├─ GeometryPool  (SoA mesh arrays)
       ├─ ILight[]      (AreaLight, DirectionalLight, DomeLight)
       ├─ IMaterial[]   (Lambertian, StandardSurface, OslMaterial, Emissive)
       └─ Camera

GeometryPool + IMaterial[]
  └─ BVHBackend::commit()
       └─ SAH BVH  (triangle list + node array, depth-first)

SceneView  (non-owning pointers to all of the above)
  └─ RenderSession::render()
       ├─ [CPU path]
       │    parallelFor(tiles)
       │      HaltonSampler (per-tile clone)
       │      BDPTIntegrator::renderTile()
       │        traceCameraSubpath()  →  PathVertexBuffer (SoA)
       │        traceLightSubpath()   →  PathVertexBuffer (SoA)
       │        connect(s,t) × bdptMISWeight()
       │          → TileBuffer (non-atomic)
       │      Film::mergeTile()  (relaxed atomics)
       │
       └─ [GPU path]
            MetalPathIntegrator::renderFrame()
              GpuMaterial[] + GpuLight[] + GpuCameraParams → MTLBuffers
              Shade.metal  (megakernel, 4-sample batch, in-register bounce loop)
              GpuAccumPixel[]  (persistent MTLBuffer)
              flushToFilm()  →  Film::mergeTile()

Film  (PixelAccumulator[], sumLumSq[], albedo AOV, normals AOV)
  ├─ [optional] Intel OIDN  →  denoised float buffer
  ├─ Film::writeEXR()       →  multi-layer HDR EXR  (OIIO)
  └─ Film::writePNG()       →  ACES tonemapped sRGB 8-bit PNG
```
